# Getting Apache / httpd Reverse Proxy to work on Fedora 30
2020-05-01

By default SELinux, in all its eternal wisdom, blocks internal loopbacks. In turn this does not allow httpd / Apache to ProxyPass to another application.


Thankfully it's a one-liner in the console to enable (you'll likely need sudo privileges):

/usr/bin/setsebool -P httpd_can_network connect 1
